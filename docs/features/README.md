# Features Documentation

This folder contains detailed documentation for each major feature or phase of the Deduper project development.

## Structure

Each numbered folder (01-20) represents a specific feature implementation or development phase:

- **01-05**: Core functionality features (file access, metadata, content analysis, etc.)
- **06-10**: Data management and performance features
- **11-15**: Advanced features and system improvements
- **16-20**: User experience and workflow enhancements

## Contents

Each feature folder typically contains:
- `CHECKLIST.md` - Implementation checklist and status
- `IMPLEMENTATION.md` - Detailed implementation guide
- `CODE_REVIEW.md` - Code review findings and notes
- `test-plan.md` - Testing strategy and plans
- `working-spec.yaml` - Working specification (for newer features)
- `change-impact-map.md` - Impact analysis for changes

## Usage

Use these documents to understand:
- How specific features were implemented
- Current status and any pending work
- Testing approaches used
- Code review feedback and decisions
